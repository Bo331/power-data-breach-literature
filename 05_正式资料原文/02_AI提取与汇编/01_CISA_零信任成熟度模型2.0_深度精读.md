### 一、 零信任顶层架构：“5 + 3”模型

CISA ZTMM 2.0 规定，零信任不能靠单一安全产品实现，必须通过 **5 大支柱** 与 **3 大横跨能力** 的交叉支撑来构建动态防御体系：

```
                    ┌─────────────────────────────────────────────────────────┐
                    │                      治  理 (Governance)                │
  ┌─────────────────┼────────────┬────────────┬────────────┬────────────┬─────┼──────────┐
  │ 3 大横跨能力     │   身 份    │   设 备    │   网 络    │ 应用与负载 │数 据│ 5大支柱  │
  │ (Cross-Cutting) │  (Identity)│  (Device)  │  (Network) │(Application│(Data│(Pillars) │
  │                 │            │            │            │ & Workload)│     │          │
  └─────────────────┼────────────┼────────────┼────────────┼────────────┼─────┼──────────┘
                    │            可见性与分析 (Visibility & Analytics)        │
                    ├─────────────────────────────────────────────────────────┤
                    │            自动化与编排 (Automation & Orchestration)     │
                    └─────────────────────────────────────────────────────────┘
```

*   **五大保护支柱 (Pillars)**：
    1.  **身份 (Identity)**：不仅指人（操作员、供应商），还包括非人实体（服务账号、API、设备凭证）。核心是“基于上下文的动态授权”。
    2.  **设备 (Device)**：泛指一切联网物理/虚拟资产（SCADA、RTU、主机、BYOD）。核心是“实时合规评估与阻断”。
    3.  **网络 (Network)**：通信介质和通道。核心是“摒弃边界防御，推行微分段（Micro-segmentation）”。
    4.  **应用和工作负载 (Application & Workload)**：私有、云端运行的程序与服务。核心是“安全交付（DevSecOps）与精细化访问”。
    5.  **数据 (Data)**：**最终的核心保护资产**。一切文件、碎片及元数据。
*   **三大横跨支撑能力 (Cross-Cutting Capabilities)**：
    *   **可见性与分析**：打通全局日志，提供策略决策依据。
    *   **自动化与编排 (SOAR)**：负责跨支柱秒级自动拦截与控制。
    *   **治理**：管理审计、法律和运营合规需求。

### 二、 数据安全支柱的演进路径（防泄露机制）

针对数据外泄（Data Leakage）防御，模型展示了数据保护如何从“人工被动防护”过渡到“智能自动拦截”：

| 成熟度阶段 | 1. 数据清单与分类标记 | 2. 存储与传输保护 | 3. 外泄检测与阻止 (DLP) |
| :--- | :--- | :--- | :--- |
| **传统阶段 (Traditional)** | 全人工、静态梳理数据，分类不完整，缺乏统一标记。 | 仅在关键节点使用静态加密。数据明文在网络中流动。 | 几乎没有外泄阻断能力，仅依赖边界防火墙做粗粒度拦截。 |
| **初级阶段 (Initial)** | 开始对新数据进行半自动分类，建立了局部的数据资产清单。 | 对部分敏感存储区加密，关键传输通道（VPN）实施加密。 | 部署基础的 DLP，能识别特定的关键字外传并产生告警。 |
| **高级阶段 (Advanced)** | 结合机器学习，跨数据源实现自动发现与分类标记，并追踪数据谱系。 | 全面实施存储加密，传输数据默认加密（如 TLS 1.3）。 | 能够识别异常的数据传输行为（如大文件外传），触发局部告警并人工阻断。 |
| **最优阶段 (Optimal)** | **全生命周期自动、动态分类**。数据产生、流转、消亡均有动态安全标签。 | 数据在生命周期的任何状态（存储、传输、**使用中**）均保持加密，动态加解密。 | **自动化联动机制**：DLP 检测到泄露风险后，SOAR 自动下发阻断策略，**零人工干预直接切断泄露链条**。 |

### 三、 核心技术：安全编排自动化与响应 (SOAR)

SOAR 是零信任架构实现**“秒级自动化响应”**的调度大脑。

#### 1. 核心三大组件
*   **安全编排 (Orchestration)**：通过 API 连接器，将不相通的安全工具（DLP、防火墙、身份认证 IAM、主机杀毒）组合连接。
*   **安全自动化 (Automation)**：将封禁、挂起、隔离、备份等人工运维步骤转化为机器自动执行的机器码。
*   **安全响应 (Response)**：下发剧本动作，快速形成闭环。

#### 2. 自动化阻断工作流：Playbook（安全剧本）示例

当电力数据安全引擎检测到某台运维主机正在外传含“敏感拓扑”的电网配置文件时，SOAR 剧本在 **5 秒内** 完成以下防御响应：

```
                    【DLP 系统捕获敏感流量告警】
                                 │
                                 ▼
                     【SOAR 引擎激活 Playbook】
                                 │
       ┌─────────────────────────┼────────────────────────┐
       ▼                         ▼                        ▼
 【身份支柱联动】            【网络支柱联动】          【设备支柱联动】
 调用 IAM 接口              调用交换机 SDN 控制器     调用 EDR 代理
 一键冻结当前操作账号。      将该主机物理端口划入      强制结束该主机上的
                            隔离 VLAN（禁网）。       涉事外发进程。
       │                         │                        │
       └─────────────────────────┼────────────────────────┘
                                 │
                                 ▼
                      【秒级拦截完成 & 留证】
                       自动捕获外发流量包留作取证，
                       并将响应结果推送给安全审计员。
```

### 四、 电力敏感数据防护的关键思考

*   **OT 环境落地冲突**：SOAR 的核心是“一键自动化阻断”（如隔离网络、挂起账号）。但在电力工业控制（OT）场景中，“阻断”可能导致电网监控中断甚至电网崩溃。
*   **折中策略**：在电力控制大区，自动化响应剧本通常需要从“全自动直接阻断”调整为“自动限制访问+自动冗余备份+人工一键确认”的半自动（Human-in-the-loop）模式。
